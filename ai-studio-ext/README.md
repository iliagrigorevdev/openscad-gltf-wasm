# AI Studio SCAD Preview Extension

A Chrome/Chromium extension that natively renders OpenSCAD (`.scad`) code blocks directly inside [Google AI Studio](https://aistudio.google.com/).

Powered by WebAssembly and Three.js, it translates OpenSCAD code to GLB on the fly, providing an interactive 3D viewer right next to your AI prompting session without needing a separate CAD application.

## ✨ Features

- **Inline 3D Previews:** Automatically detects `openscad` / `scad` code blocks generated by the LLM and injects a "👀 Preview 3D" button.
- **Advanced SCAD Prompt Generator:** A built-in UI overlay that instantly generates and injects detailed system instructions teaching the AI how to use modern PBR materials and animations based on your description.
- **Client-Side Compilation:** Uses [`openscad-gltf-wasm`](https://github.com/iliagrigorevdev/openscad-gltf-wasm) to securely compile SCAD scripts completely in the browser.
- **Advanced 3D Rendering:** Built-in Three.js viewer with orbit controls, auto-camera framing, and Image-Based Lighting (IBL).
- **PBR & Animation Support:** Fully supports the extended PBR materials (metalness, roughness, glass transmission) and skeletal animations provided by the custom OpenSCAD fork.
- **Local Saving:** Seamlessly save AI-generated `.scad` code directly to your local machine via a local backend server.

## 🚀 Getting Started

### Prerequisites

- Node.js (v16+ recommended)
- NPM or Yarn

### 1. Build the Extension

This project uses Vite to bundle the Three.js and WASM dependencies.

```bash
# Install dependencies
npm install

# Build the extension for production
npm run build
```

_(The built extension files will be output to a `dist/` folder)._

### 2. Load into Chrome

1. Open your browser and navigate to `chrome://extensions/`.
2. Turn on **Developer mode** (toggle in the top right).
3. Click **Load unpacked**.
4. Select the output directory (e.g., `dist/`) containing the compiled `manifest.json`, `content.js`, and HTML assets.

## 🧪 Local Testing (Standalone)

If you want to test the 3D renderer or tweak the Three.js lighting without loading the extension into AI Studio, a standalone tester page is included:

```bash
npm run dev
```

Open the provided `localhost` URL in your browser. You will see a split-screen layout with an SCAD editor on the left and the 3D preview iframe on the right.

## 🛠 Usage in Google AI Studio

### 🪄 1. Generating Advanced Prompts

Because standard LLMs are mostly trained on legacy OpenSCAD syntax, they don't natively know how to generate PBR materials or animations. The extension provides a tool to bridge this gap:

1. Open [Google AI Studio](https://aistudio.google.com/).
2. Click the floating **✨ SCAD Prompt** button in the bottom-left corner of the window.
3. Toggle the advanced features you want the AI to utilize (e.g., Basic PBR, Transmission, Emissive, Animations).
4. Enter a brief description of what you want (e.g., _"A shiny gold ring with an embedded red gem"_).
5. Click **📋 Generate & Paste**. The extension will automatically craft the complex rules, inject them into your AI Studio chat box, and copy them to your clipboard!

### 👀 2. Previewing 3D Models

1. Submit your prompt and wait for the model to start generating the OpenSCAD code.
2. As soon as the code block appears, a **👀 Preview 3D** button will be injected at the top of the snippet.
3. **⚠️ Note:** Wait until the AI has completely finished generating the code before clicking the button! Trying to preview while the code is still streaming will result in syntax errors and a broken render.
4. Once the generation is complete, click the button to open the floating 3D viewer.
5. Drag to rotate, scroll to zoom. Click the **X** to close the overlay.

### 💾 Saving Models Locally

The 3D preview overlay includes a Save UI in the top-left corner, allowing you to instantly save the generated `.scad` code to your computer.

Because browser extensions cannot write directly to your file system silently, this feature connects to a local development environment using the `scad-serve` CLI.

**1. Start the local backend server:**
Run this in your desired save folder to bridge the extension to your filesystem (runs on port `3000` by default).

- **Option A: Run directly (No installation)**
  ```bash
  npx -p github:iliagrigorevdev/openscad-gltf-wasm scad-serve
  ```
- **Option B: If installed locally (`npm install --save-dev github:iliagrigorevdev/openscad-gltf-wasm`)**
  ```bash
  npx scad-serve
  ```

**2. Save from the Extension:**

1. Once the AI generates code, open the **👀 Preview 3D** window.
2. In the top-left corner, enter your desired filename (e.g., `my_model`).
3. Click **Save**. The extension will check if the file already exists (prompting for overwrite if needed) and save the `.scad` file directly to the folder where you ran the server.

## 🧠 Better Prompts with AI (Under the Hood)

The UI generator replaces the need to manually script context blocks. However, if you are building your own tools, the extension relies on the `generatePrompt` API from the core library to dynamically build syntax instructions for the LLM:

```javascript
import { generatePrompt } from "openscad-gltf-wasm/prompt";

// The extension UI calls this under the hood and pastes the result:
const instructions = generatePrompt("a shiny red sports car", {
  basic: true,
  clearcoat: true,
  animation: false,
});
console.log(instructions);
```

## 📁 Architecture

- **`content.js` / `content.css`**: Content scripts injected into AI Studio. Monitors the DOM for `ms-code-block` elements to add the 3D preview overlay, and manages the SCAD Prompt modal injection and auto-pasting logic.
- **`preview.html` / `preview.js`**: The sandboxed iframe environment containing the Three.js scene and Emscripten WASM compiler. Isolated to safely handle `wasm-unsafe-eval`.
- **`openscad-gltf-wasm`**: The underlying engine doing the heavy lifting of converting `.scad` text into `.glb` binary buffers and providing the prompt schemas.
